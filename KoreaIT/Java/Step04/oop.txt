
	절차지향언어
		일을하는 순서에 초점을 맞춘 언어 --> C, 어셈블리 등
	객체지향언어
		일을 하는 대상을 표현(코드)
		
		1. 캡슐화(Encapsulation)
		
		2. 상속(Inheritance)
		
		3. 다형성(Polymorphism)
		
		
	클래스
		표현하려는 대상을 표준화하여 코드로 작성한 것
		클래스 구성 요소
			필드 : 저장할 데이터를 담당, 변수선언
			메서드 : 기능적인 부분을 담당, 메서드 선언
			생성자 : 객체 생성시 해야되는 첫번째 기능
			
	접근제어자
		private : 클래스 내부에서만 접근 가능하다
		protected : 상속받은 클래스까지 접근이 가능
		default : 같은 패키지 내에 있으면 접근이 가능  <-- 접근제어자를 안쓴경우는 다 default 라고 함. 
												같은 폴더 안에 있으면 접근가능
		public : 외부에서도 사용가능
		
		public void /public 생략가능
